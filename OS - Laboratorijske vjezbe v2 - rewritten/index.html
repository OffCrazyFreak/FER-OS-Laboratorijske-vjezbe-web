Postupak za vježbe


  1. Pročitati pripremu za vježbu i samostalno riješiti programske zadatke (domaću zadaću).
  2. Prijaviti termin u tjednu predaje vježbi putem sustava ferko.
  3. Programski kod arhiviran u .zip formatu prema uputama postaviti i zaključati na sustav ferko do utorka do 15 sati u tjednu prijavljenog termina.
  4. U zakazanom terminu, asistentu demonstrirati rad programa te po potrebi odgovoriti na pitanja.

Prijava termina

Prijave započinju tjedan dana prije termina labosa, a završavaju početkom odabranog termina ili kada se popune svi raspoloživi termini.

Svaki nedolazak na prijavljeni termin labosa kažnjava se negativnim bodom.

Demonstracija rada

U nekom terminu je moguće demonstrirati rad najviše dvije vježbe. U terminu nadoknade moguće je demonstrirati rad najviše jedne vježbe.

Vježbe se predaju po redu (osim u iznimnim slučajevima kada se zbog opravdanih razloga predaje zaostala vježba u terminu nadoknade).

Domaća zadaća

× 
#include <time.h>
#include <math.h>
#include <unistd.h>
#include <stdlib.h>
          
int random_prime()
{
  static unsigned int seed = -1;
  unsigned int i, a, x, max_q;
  
  if (seed == -1)
    seed = time(NULL) + getpid();

  while ((x = rand_r(&seed)) < 10)
    ;

  x |= 1; /* make him odd if its not */

  max_q = (int) sqrt((double) x);

  for (i = x; ; i += 2) {
    for (a = 3; a <= max_q; a += 2)
      if ((i % a) == 0)
        break;
    if (a > max_q)
      break; /* 'i' is prime */
  }

  return i;
}

*
* U vj2.c negdje ispod 'include'-ova dodat:
*   extern int random_prime();
*
* Prevodjenje:
*  gcc vj2.c random_prime.c -lm
*
* Pokretanje:
*  ./a.out 5 10
*/#include <stdio.h>
#include <signal.h>

void prekidna(int sig)
{
  printf("U obradi prekida\n");
  sleep(5);
  printf("Obrada prekida gotova\n");
}

int main()
{
  struct sigaction act;
  
  act.sa_handler = prekidna;
  sigemptyset(&act.sa_mask);
  sigaddset(&act.sa_mask, SIGQUIT);
  act.sa_flags = 0;
  
  if (sigaction(SIGINT, &act, NULL)) {
    fprintf(stderr, "Nisam postavio masku za signale\n");
    perror("");
    exit(1);
  }

  while(1) ;
  
  return 0;
}

Programski kod pisati prema uputama (posebice paziti na uvlačenje, razmake te prijelom dugih linija). Primjeri korištenja stila prikazani su u kodovima random_prime.c i sigaction.c. Neki od priloženih programa koriste neznatno modificirani stil (više razmaka) koji je također prihvatljiv.


Sve datoteke koje sadrže izvorni programski kod koji čine jednu vježbu (pa makar bila i samo jedna datoteka) staviti u .zip arhivu.

Sustav za detekciju plagijata ne prihvaća druge arhive osim .zip (primjerice .rar, .gz i sl.)!

Nikakve druge datoteke ili direktorije ne stavljati u arhivu (primjerice arhiva ne sadrži makefile)!

Odabrati pravu datoteku za upload je izazov i boduje se, odnosno upload pogrešne datoteke u detekciji plagijata se kažnjava negativnim bodovima.

Učestale pogreške koje nose negativne bodove su: korištenje neke druge arhive osim .zip, predaja vježbe na krivom mjestu, arhiva sadrži poddirektorije ili izvršne datoteke.

Moguće je otključati vježbu koja je greškom predana i zaključana za jedan negativan bod slanjem email-a sa zahtjevom na os@zemris.fer.hr!

Dodatno

Sve vježbe se moraju postaviti na sustav ferko najkasnije do ponedjeljka, 06.06.2022. do 15 sati.

Studenti koji su ranijih godina odradili laboratorijske vježbe, ali nisu položili predmet, moraju odraditi i ovogodišnje laboratorijske vježbe.

Žalbe su moguće tek na kraju semestra kad su objavljeni ukupni rezultati.
